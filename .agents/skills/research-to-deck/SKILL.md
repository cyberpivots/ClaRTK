---
name: research-to-deck
description: Use when ClaRTK needs a verified development summary converted into a repo-native slide source and Canva publication brief.
---

1. Start from tracked evidence only: task files, ADRs, docs, generated artifacts, and checks actually recorded or run in the current task.
2. Pull external guidance only from primary or official sources, and record the source URLs plus relevant dates.
3. Reject or defer any claim that lacks a repo file path, check result, or verified external citation.
4. Produce both artifacts under `docs/presentations/`: a versioned slide source and a Canva brief with a slide-by-slide source map.
5. Keep the repo deck canonical. Any Canva output is derived and must preserve repo facts verbatim.
